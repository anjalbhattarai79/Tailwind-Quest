Git Unadd Guide

When you've added files to Git's staging area with git add but want to remove them from the staging area before committing, you need to "unadd" them.

Git Workflow

1

Working Directory

Your files before staging

2

Staging Area

Files added with git add

3

Repository

Files committed with git commit

Unadding files moves them from the staging area back to the working directory.

Commands to Unadd Files

Command Description When to Use
git reset HEAD <file> Unstages a specific file When you want to remove one file from staging
git reset HEAD . Unstages all files in current directory When you want to remove all files in current folder from staging
git reset HEAD Unstages all files When you want to remove all files from staging
git restore --staged <file> Unstages a specific file (Git 2.23+) Newer alternative to git reset

Examples

# Unadd a specific file
git reset HEAD index.html

# Unadd multiple files
git reset HEAD style.css script.js

# Unadd all files in current directory
git reset HEAD .

# Unadd all files in repository
git reset HEAD

# Using git restore (Git 2.23+)
git restore --staged image.jpg

Important: git reset doesn't delete your files or changes. It only removes them from the staging area. Your modifications will still be present in your working directory.

Checking Status

Always use git status to see what files are staged before and after unadding:

git status

This will show you which files are staged (ready to be committed) and which are not staged.
